Block

俄罗斯方块，就是小方块构成的游戏。
小方块——Block或者Cell,是游戏中最小，也最根本的元素。
因为他将要构成一切：

  1. 单个小方块，即自己。
  2. 一个4X4小方块组，通过不同的显示和排列构成Tetromino——即形成0,I,S,Z,T,L,J 7种俄罗斯方块。
  3. 一个10X20小方块，即构成了整个Playfield游戏区。

所以关于1

  1. 小方块需要知道自己组成了谁的头部，我的意思是：我的老大是谁：Tetromino,还是Playfield.
  2. 小方块需要有2种状态，记录自己是否需显示，也就是occupied 还是 empty。
  3. 小方块需要切换自己的这2种状态，并根据状态来显示。
  4. 小方块需要知道自己在老大中的位置。（因为其是按序排列的）

掌握了这些信息，其他的就引刃而解了，老大和老二（Playfield,Tetromino)所有操作都交给组成它们的这些小方块就行了。

所以要解决的问题其实不多。就是小方块。
难怪很多前辈都推荐写俄罗斯方块入门，一个比较简单但是又有趣味性的游戏，不需要什么图片资源，开发逻辑简单，但是又几乎包含一款游戏的所有元素和流程。

首先定义关于Block的结构体

BlockDef包含了初始化一个小方块必要的信息，将在Block实例化时作为参数传入。
Block对象还实现了切换自身状态，获取自己位置,移动判断等方法。

GhostBlock ,BlockElement将继承这个Block类，分别实现
virtual std::string getBlockFileName() = 0;以显示不同的图片，
另外它们也将分别实现自己的方法。
BlockElement将用来构成7种方块和playfield，而GhostBlock则将构成GhostPiece跟随当前活动的方块，提供一个最终位置的效果。

Tetromino

由小方块构成

  1. 1个定时器根据速度等级来进行softdrop
  2. 根据按键移动位置
  3. 旋转
  4. 当不能移动时(到底时），锁定位置，关闭计时器
  5. 与Playfield交换信息，让playfield根据自己的方块和位置进行填充

SRS 旋转

Super rotation system.听起来好高大上。

这是官方指导中SRS的旋转方案。
本来想直接用cocos2dx的旋转函数，每次点击旋转即旋转90度来解决，但是发现这7种方块中，有些锚点在方块交界，有些在方块正中。感觉这样会增加方块锁定时，与游戏区域(PlayField）小方块之间坐标转换的复杂度。
一切从简，毕竟是新人。
采用的旋转方案是多数自制过俄罗斯方块的人都采用的预先硬编码方式。毕竟每种方块都仅有4种形态。预先硬编码还是蛮高效的。将每个方块都定义成4X4的结构，用0、1分别表示该点是否被占用。所以，可以用1个char16_t来存储方块的每一种形态，用一个char16_t[4]数组来预存储方块的4种对应旋转姿态。




每次旋转操作就是：

  1. 读取char_16[4]数组中下一个元素
  2. 位移操作比较当前块的占用状态是否发生变化
  3. 发生变化则重绘当前对应小方块。

移动和旋转操作

其实就是对每一个小方块进行移动和旋转判断，如果成功就移动，失败就返回而已啦。
为了提高效率:左移就从最左边行列开始检测，右移就从最右边开始，下移就从最下列开始，出现失败情况，立刻返回。

Playfield

也是一个10X20方块宽高的游戏面板，也可以理解为是极端情况下被10X20个小方块填充的区域。被不同颜色的小方块填充就显示不同的颜色。所以说到底还是方块。
我曾到stackoverflow上查问关于俄罗斯方块实现的问题，看到一些关于是playfield独立由数组填充还是免去playfield而由各独立piece填充的问题。但个人感觉，还是由10X20的playfield来实现方块填充来的直接。首先，这本就是一个矩阵问题，可以更简单的判断行满（进行消行），列满（进行游戏结束）等的判断。也可以更方便的把它构想成1个用来接纳各游戏方块的容器。
所以还是就用这种简单明了的方式吧。缺点可能是比较耗内存吧。


Softdrop and Harddrop

soft drop定义一个根据速度等级变化的timer定时更新方块的下落位置即可。
hard drop直接将方块位置设定为当前GhostPiece的位置。

Random generator.

random generator.随机方块，Tetris Guideline中提出的随机方案是“包”随机，即每次打乱这7个方块的序列包，从包中取出打乱顺序的方块，取完后在放回包中，重打乱。给人一种是从1个有7个方块的包裹中伸手依次取出一个方块直到取空，然后放回摇晃后，再取的感觉。当然，这是我的理解。希望没理解错吧。如果没错的话：
恩，一个random_shuffle就可以解决了。

Wall Kick.

这个在官方SRS系统中就给出方案了。就照它的写吧。

Hold Piece.

暂存块。玩家点击暂存后：

  1. 暂存块空，按当前块形式新创建一个暂存快，让下一块（next piece)入场。
  2. 已有暂存块， 将当前块切换为暂存快，暂存快切换为当前块。开始当前块soft drop.

Ghost Piece.

  1. 定位至当前块能到达的最低端位置
  2. 切换为当前块的形状（type)

Next Piece.

从random generator中取出下一个将来到的元素即可。
