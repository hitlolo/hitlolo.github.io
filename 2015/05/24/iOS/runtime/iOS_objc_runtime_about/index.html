An object-oriented approach to application development makes programs more intuitive to design, faster to develop, more amenable to modification, and easier to understand. Most object-oriented development environments consist of at least three parts:
● A library of objects
● A set of development tools
● An object-oriented programming language and support library

IOS Developer LibraryObject-Oriented Programming with Objective-C

以上提到面向对象开发环境至少由3部分组成：

  • A library of objects
    这个是 Cocoa丰富的 Framework.
  • A set of development tools
    这个是 Xcode,Clang,Instruments...
  • An object-oriented programming language and support library
    语言和支持库，Objective-C和Runtime。

Language and Support Library

这里的“support library”我理解的是Runtime Library。
运行时系统是Objective-C的关键，Objective-C语言的动态特性和面向对象功能都由它来完成，被视作 Objective-C的操作系统。

The Objective-C language defers as many decisions as it can from compile time and link time to runtime. Whenever possible, it does things dynamically. This means that the language requires not just a compiler, but also a runtime system to execute the compiled code. The runtime system acts as a kind of operating system for the Objective-C language; it’s what makes the language work.

runtime的工作过程包含两个阶段：

  • 编译（compile time）
  • 运行（run time）

    • 在编译阶段：编译器利用runtime library（compile time）生成用于运行阶段数据结构，给runtime system提供运行时支持。

    在runtime阶段：Objective-C运行时系统（runtime system）根据编译器在编译阶段提供的数据结构和方法为程序提供运行时支持。
