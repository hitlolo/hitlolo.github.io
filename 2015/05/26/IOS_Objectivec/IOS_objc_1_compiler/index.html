预处理

编译器要的第一个阶段是预处理。调用的工具是预处理器。
跟预处理器打交道的是预处理语言，预处理器将根据预处理指令进行代码替换以及宏展开，以此完成对代码的第一阶段处理。
查看预处理器的输出结果可以使用命令行：
clang -E main.m
也可以Xcode：Product/Perform Action/Preprocess main.m
可以看到输出文件中

编译

这个阶段编译器接收预处理后的文件，进行词法，语法，语义分析然后完成类型检查，代码优化等工作，最后输出为中间代码，如汇编。以完成进入下一步的准备工作。

词法分析（Lexcical Analysis）

编译器将源文件中的定义，声明，关键字等分解成一个个看似无逻辑关系的标示符，完成初步分解。
使用编译器命令：
clang -Xclang -dump-tokens main.m

语法分析（Syntax Analysis）

语法分析阶段，编译器将接收词法分析器的输出结果，通过语法分析器创建树形的中间表示AST（Abstract Syntax Tree)。AST给出了源代码的语法结构，对后端的处理提供一个比较清晰的接口。编译器的后续步骤将根据AST来分析代码。
clang -Xclang -ast-dump main.m
输出

根据AST，编译器掌握了代码的语法结构和信息，将以此为基础作语义分析。

语义分析（Semantic Analysis）

语义分析根据AST和符号表来分析源代码与语言定义的语义是否一致。
包括：静态分析，类型检查，也可进行代码改写与代码优化。
命令：
clang -fsyntax-only
clang -analyze
可查看相应分析的结果。

  2. _cmd 当前方法的selector

在阶段编译器将进行代码改写，把Objective-C的数据结构，消息传递改写以生成使用了Objective-C运行时系统库的代码，以配合运行时runtime system的合法执行。

代码改写

一切分析通过后，编译器利用此时的AST进行代码改写和中间代码生成，转换为中间码LLVM IR。
要查看这个过程的输出 可以使用命令
clang -rewrite-objc 改写为C代码
clang -S -emit-llvm 生成LLVM IR

汇编

最后编译器将代码优化后，将其汇编成为汇编代码。
汇编代码可以由
clang -S命令查看。
得到Assembler。
整体上看上去还是很眼熟的。
可以体会到从源代码->AST->改写C->llvm->assembler的痕迹。
编译器还剩下的最后一个翻译步骤是将现在的汇编代码汇编成目标的机器码，生成目标文件”.o”。
可以使用命令：
clang -c得到汇编后的输出”.o”。文件中的内容是机器码，只能使用反汇编查看其中的内容。
命令nm -nm可以得到目标文件中的符号表。
符号表是编译器用来保存源代码构造的各种信息的数据结构。这些数据在分析阶段被逐步收集，符号表每个条目都包括与一个标示符相关的信息，如一个函数名字，它的参数数量，参数类型，参数的传递方法，返回信息等信息。

链接

汇编后的目标文件与与其他需要的目标文件链接以生成可执行文件。
如示例程序需要与Foundation框架链接以生成可执行程序。
执行命令：
clang -framework Foundation main.o
此时在查看符号表
nm -nm a.out
得到：

经过链接后新生产的符号表已经发生了变化：

5-8行：

_objc_empty_cache，_objc_msgSend 都(from libobjc)
这里的libobjc正是运行时系统所在的库，这些服务都来源于Runtime, runtime system也正是在链接阶段动态链接进来的。

至此，万事俱备：
./a.out
终于看到了hello world.
源->tokens->AST->改写->llvm->object code确是漫漫长路。

从上面可以看到，编译器将Objective-C代码改写了特别的C数据结构和方法以配合Runtime来支持其Objective的特性。
